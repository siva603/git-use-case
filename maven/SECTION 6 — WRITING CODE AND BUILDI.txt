1. Add Java Source File

In a Maven project, application code must be placed in the following directory.

src/main/java

Example directory structure

project
pom.xml
src
src/main/java

Create package structure

mkdir -p src/main/java/com/example

Explanation

mkdir → create directory
-p → creates nested directories automatically

---

2. Create Java Class

Create a Java file inside the directory.

Example

src/main/java/com/example/App.java

Example code

public class App {

```
public static void main(String[] args) {
    System.out.println("Hello Maven");
}
```

}

Explanation

public class App → class definition

public static void main → entry point of Java program

System.out.println → prints output to console

---



3. Compile the Project

Compile the Java source using Maven.

Command

mvn compile

Explanation

mvn → Maven command tool
compile → compiles Java source code

What Maven does internally

1. Reads pom.xml
2. Finds source files inside src/main/java
3. Compiles them
4. Stores compiled classes in target directory

---

Output Directory

After compilation

target/classes

Example

target/classes/com/example/App.class

Explanation

.class → compiled Java bytecode

---

4. Maven Compiler Plugin

Maven uses a plugin to compile Java code.

Plugin name

maven-compiler-plugin

Example configuration in pom.xml

<build>
  <plugins>

```
<plugin>
  <groupId>org.apache.maven.plugins</groupId>
  <artifactId>maven-compiler-plugin</artifactId>
  <version>3.11.0</version>

  <configuration>
    <source>17</source>
    <target>17</target>
  </configuration>

</plugin>
```

  </plugins>
</build>

Explanation

source → Java source version
target → generated bytecode version

Recommended modern version

Java 17 or Java 21

---

5. Run the Application

After compilation you can run the program.

Command

java -cp target/classes com.example.App

Explanation

java → Java runtime

-cp → classpath option

target/classes → location of compiled classes

com.example.App → fully qualified class name

Example

java -cp target/classes com.example.App

Output

Hello Maven

---

6. Package the Application

Create executable artifact.

Command

mvn package

Explanation

package → compiles code and creates artifact

Output

target/project-name.jar

Example

target/demo-project.jar

---

7. Update Project in Git

After adding code, update the repository.

Check changes

git status

Add files

git add .

Commit changes

git commit -m "Added Java source and build configuration"

Explanation

status → shows changes
add → stages files
commit → saves snapshot

---

Typical Development Workflow

Write code
↓
Compile project

mvn compile

↓
Run tests

mvn test

↓
Package project

mvn package
