# Docker — Complete Learning Guide (Beginner → Advanced)

# Section 1 — Introduction

## What is Docker

Docker is a **containerization platform** used to package applications along with their dependencies into **portable units called containers**.

A container ensures that an application runs the **same way on every machine**.

Example problem before Docker:

Developer machine → Works  
Testing server → Fails  
Production server → Different behavior

Docker solves this problem.

---

## Why Docker Exists

Traditional software deployment had many issues:

• Dependency conflicts  
• Different OS environments  
• Hard to scale applications  
• Difficult to reproduce bugs

Docker solves these problems by packaging:

Application  
Libraries  
Runtime  
Configuration

inside **containers**.

---

## Containers vs Virtual Machines

### Virtual Machine

VM includes:

Application  
Libraries  
Guest OS  
Hypervisor

This makes VMs **heavy and slow**.

### Containers

Containers include:

Application  
Libraries  
Runtime

Containers **share the host OS kernel**, making them:

• Lightweight  
• Fast  
• Portable

---

## Docker in DevOps Pipeline

Typical modern pipeline:

Developer  
↓

↓  
Docker Build

↓

Docker acts as the **standard packaging format for applications**.

## Key Docker Features

• Lightweight virtualization  
• Fast application deployment  
• Version-controlled images  
• Easy scaling

# Section 4 — Core Concepts

## Docker Architecture

Main components:

Client → Docker CLI
Daemon → Docker Engine
Registry → Docker Hub

Workflow:

Developer runs command
↓
Docker Client sends request
↓
Docker Engine executes
↓
Image pulled from Registry

---

## Docker Engine

Docker Engine manages:

• Containers
• Images
• Networks
• Volumes

It runs as a **background service**.

---

## Docker Images

A Docker image is a **read-only template** used to create containers.

Images contain:

• OS base layer
• Runtime
• Libraries
• Application code

Example:

Ubuntu Image
Node Image
Python Image

---

## Docker Containers

A container is a **running instance of an image**.

Example:

```
docker run ubuntu
```

This creates and runs a container.

---

## Dockerfile

A Dockerfile defines **how to build an image**.

Example:

```Dockerfile
FROM node:18
WORKDIR /app
COPY . .
RUN npm install
CMD ["node","server.js"]
```

Explanation:

FROM → Base image
WORKDIR → Working directory
COPY → Copy files
RUN → Execute command
CMD → Default start command

---

## Docker Image Layers

Images are built using **layers**.

Example:

Layer 1 → Base OS
Layer 2 → Runtime
Layer 3 → Dependencies
Layer 4 → Application

Layer caching makes builds **fast**.

---

## .dockerignore

Used to prevent unnecessary files from entering images.

Example:

```
node_modules
.git
.env
```

# Section 5 — Core Workflow

## Pulling Images

Download image from registry.

```bash
docker pull ubuntu
```

---

## Running Containers

Start container:

```bash
docker run ubuntu
```

Run interactive shell:

```bash
docker run -it ubuntu bash
```

Options:

* `-i` interactive
* `-t` terminal

---

## Detached Mode

Run container in background.

```bash
docker run -d nginx
```

---

## Container Port Binding

Expose container port.

```bash
docker run -p 8080:80 nginx
```

Meaning:

Host port → Container port

---

## List Containers

Running containers:

```bash
docker ps
```

All containers:

```bash
docker ps -a
```

---

## Stop Container

```bash
docker stop container_id
```

---

## Remove Container

```bash
docker rm container_id
```

---

## Remove Images

```bash
docker rmi image_name
```

# Section 6 — Docker Storage

Containers are **ephemeral**.

Data disappears when container stops.

Docker provides storage solutions.

---

## Volume Mounts

Volumes store persistent data.

Create volume:

```bash
docker volume create mydata
```

Mount volume:

```bash
docker run -v mydata:/data ubuntu
```

---

## Bind Mounts

Mount host directory.

```bash
docker run -v /host/path:/container/path
```

Used for development.

---

## Tmpfs Mounts

Stores data in **memory only**.

Used for temporary sensitive data.

---

# Section 8 — Docker Compose
